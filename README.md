## FinanceAnalytics API

**FinanceAnalytics API** — backend-модуль для анализа банковских транзакций, формирования финансовой аналитики 
и генерации JSON-ответов для веб-страниц.

Проект обрабатывает операции из Excel-файла, анализирует расходы и доходы, рассчитывает кешбэк, формирует 
агрегированные данные по категориям, получает курсы валют и стоимость акций через внешний API и возвращает 
структурированные JSON-ответы.

### Основные возможности

#### Аналитика транзакций

- Фильтрация по дате (день / неделя / месяц / год)

- Подсчет общей суммы расходов

- Подсчет общей суммы доходов

- Топ-5 транзакций по сумме

- Топ категорий расходов

- Анализ поступлений по категориям

- Отдельный расчет расходов по:

    - «Переводы»
    
    - «Наличные»

- Анализ трат по выбранной категории за последние 3 месяца

#### Работа с кешбэком

- Расчет самых прибыльных категорий кешбэка за выбранный месяц

- Группировка по категориям

- Возврат результата в JSON-формате

#### Поиск по транзакциям

- Поиск по описанию или категории

- Поиск переводов физическим лицам (по шаблону Фамилия И.)

- Возврат результата в JSON

#### Интеграция с внешними API

- Проект использует API сервиса:

    - Twelve Data

- Получаемые данные:

    - Курсы валют к рублю
    
    - Стоимость выбранных акций (S&P500)

- API-ключ передается через переменную окружения:
```
API_KEY_TWELVEDATA
```

### Архитектура проекта

Проект разделен на логические модули.

_1. **Работа с данными**_

read_xlsx.py

- Чтение Excel-файла

- Преобразование дат в datetime

- Возврат pandas DataFrame

_2. **Views** (формирование JSON для веб-страниц)_

**Страница «Главная»**

Функция:

get_information_home_page()

Возвращает JSON со структурой:
```
{
  "greeting": "...",
  "cards": [...],
  "top_transactions": [...],
  "currency_rates": [...],
  "stock_prices": [...]
}
```

**Страница «События»**

Функция:

get_events_information()

Формирует:

- расходы

- доходы

- топ категорий

- переводы и наличные

- валюты и акции

**_3. Services_**

get_transactions_list

get_profitable_cashback_categories

make_simple_search

search_for_transfers_to_individuals

Бизнес-логика вынесена отдельно от представления.

**_4. Reports_**

Декоратор @report

Позволяет:

- логировать выполнение функции

- сохранять результат в файл

- фиксировать ошибки

Пример:
```
@report(filename="reports.txt")
def get_spending_by_category(...)
```

_**5. Logging**_

Универсальный конфигурируемый логгер:

setup_logger()

Особенности:

- автоматическое определение имени модуля

- логирование в файл

- поддержка консольного вывода

- очистка логов при старте

- гибкая настройка формата

Логи сохраняются в директорию:

/logs

### Установка
```
git clone https://github.com/simbarilion/Banking_transaction_analyzer
cd FinanceAnalytics
poetry install
poetry shell
```

#### Настройка API

Создайте файл user_settings.json:
```
{
  "user_currencies": ["USD", "EUR"],
  "user_stocks": ["AAPL", "MSFT"]
}
```
Добавьте переменную окружения:
```
API_KEY_TWELVEDATA=your_api_key_here
```

#### Запуск
```
python main.py
```

#### Пример вывода

Программа выводит JSON:

- данные для главной страницы

- события

- прибыльные категории кешбэка

- результаты поиска

- переводы физлицам

- отчет по категории
